some things I have made

  • Lawfetcher

    A totally client-side app to parse US legal citations and program­matically build direct links to the source document being cited. The project's was adopted to an institutional home under the aegis of Cornell Law's Legal Information Institute and rebranded as RIO.

    Technically, the project has been an excellent vector for learning about testing, build scripts, functional programming, and javascript transpilation. It has also provided a fascinating, albeit limited-scope, high-level overview of the structure of US law.

  • Wedding Photo Gallery

    As my wife and I were looking through the smugmug gallery of our wedding photos, the slideshow UI kept locking up. I decided making the photo list a doubly linked list would prevent that sort of memory problem, and rolled my own gallery.

    There's a rails backend that handles zipping sets of full-resolution photos for download. Rails isn't a perfect choice for such a small backend, but a) I wanted to practice testing in a rails environment, and b) this setup will be easier to extend down the line with features like tagging people and filtering the photo list by last or full names.

  • Sudoku Solver

    A sudoku solver written just ten weeks after starting Dev Boot­camp. (Okay, I dressed up the css a bit afterwards, but the logic is very much a reflection of me as a rank beginner.) Can solve any puzzle (that is possible to solve), even ones that require guessing. DOM interactions done with jQuery; solving logic all in vanilla ES5.

  • Buckets

    Here's the pull request I made for API inter­actions and data storage for an NBA stats app. The app's development was arrested at the proof of concept stage, once I read the small print and saw it unambiguously stated that outside use of the stats.nba.com API (at least circa when I was working on it) wasn't legal; but it was a good exercise in trying to write an Adapter that isn't horrible and a thoroughly-documented PR.
