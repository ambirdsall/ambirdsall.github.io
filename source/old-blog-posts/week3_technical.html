JavaScript in Web Development

Alex Birdsall
6.2.2014

At the time of writing, JavaScript has become pretty ubiquitous in web development. Blog comments, interactive elements, styling that changes with user interactions, rotating slideshows: almost anything that moves on a website moves with JS. It's a contender for most popular language amongst GitHub users. But why is it that JS, amongst all the languages of technobabel, has come to dominate the web?

For starters, JS is structurally pretty well-suited to use in websites. Programming languages interact with websites via the Document Object Model; JS is an object-oriented language. Specifically, it's a prototype-based language, which allows programmers to build their own objects, structure them how they please, and then build new objects that inherit those features. That means that JS is well-suited to making flexible, ad hoc structures suited to specific situations. Since websites are structured through nested tags that can be used in idiosyncratic ways, a language like JS that allows you to create "bespoke" methods and data structures suited to the site structure at hand is ideal.

Still, there are plenty of object-oriented languages that could have served. The biggest advantage JS holds over the others is, quite simply, inertia. JavaScript was released in 1995, and a large community has grown in its 19-year history. There are many libraries of prewritten JS designed for use on web projects. There is also AJAX, a set of techniques to update parts of a website without having to reload. It's possible to replicate the effects of AJAX with another language, but why bother when the code, standards, libraries and frameworks for AJAX are already written? With JavaScript, your code can go further by standing on the shoulders of giants.

And just as the resources available to JS developers have proliferated, so have the standards of implementation. Every major browser supports JS, which is run in a sandbox within the browser. This local implementation is rather faster than constant communication with a remote server, and it has allowed for a series of limitations to be imposed on JS for security. A website's JS cannot access files on your computer or on a server, and JS cannot access information about any website on a different domain. This helps prevent the spread of malignant code.

There are downsides to JS, of course. The ability to access files or databases directly, rather than through calls to separate, server-side scripts, would allow websites to manipulate data faster and more powerfully. And in addition to its security-motivated limitations, JS has no graphical functionality. It can alter text and CSS styling, but it can't directly create and manipulate images or animations.

So there you go. While it has its shortcomings, JavaScript is awfully well-suited to use in websites, and has organically grown a large community and a really useful set of resources over the years. Taken as a whole, this makes it about the easiest and most effective tool to make a website do whatever you want it to do. This self-reinforcing feedback loop of usefulness and popularity has established JavaScript as the web's language of choice.
