Ruby on /Regexp/

Alex Birdsall
7.6.2014

Regular expressions, often abbreviated "regexp", are sets of characters and metacharacters that form search patterns. Each piece of these patterns can be either a specific character or a set of characters, and each can be given quantitative specifications. Once composed, regular expressions are compared to strings in order to find any set of characters that matches the given pattern, but which may vary in their particulars. For instance:

  • any arbitrary phone number, such as "(555) 555-5555" or "555-555-5858"
  • a word with multiple valid spellings ("color" vs "colour"; "Ã–zil" vs "Oezil"; "grey" vs "gray")
  • every instance of a formatting convention such as douple spaces after a sentence-ending punctuation mark, followed by a capital letter

Regular expressions have been in use since the late 60s, and most programming languges have regexp capabilities, either as part of the language or through a standard library. This post centers on regular expressions in Ruby; most, but not all, of this applies to other languages as well.

There are three ways to create a regular expression in Ruby: using the literals /some regexp/ and %r{some regexp} and the constructor some_regexp = Regexp.new. Which means that Ruby has a built-in Regexp class, with methods. The fundamental method, RegExp#match, takes a string as a parameter and does exactly what it sounds like, returning a MatchData object. Typically, though, the =~ operator is used: a regexp goes to the left of the operator, a string to the right, and the whole expression returns either the index of the first character of the first match within the string or nil.

The simplest regular expressions are simply lists of characters, like /dog/. Any string that contains those exact characters in that order, including spaces, will return a match, as within: "hot dog, hotdog, dogpile, Dogpatch<"/span>.

Characters within brackets are called "character classes": character classes any single character listed within the brackets. For example: /d[oi]g/ =~ "dog, dig, doig, diog, dogpile" will find the matches "dog, dig, doig, diog, dogpile". You can use ranges within character classes: /[a-e]/ is equivalent to /[abcde]/, and /[a-eW-Z]/ is equivalent to /[abcdeWXYZ]/. Putting a ^ at the start of a character class negates it: /[^a-e]/ matches any letter EXCEPT "a", "b", "c", "d", or "e".

Outside of a character class, that caret has a different special meaning. These special characters, called "metacharacters", are what give regular expressions their expressive power. Non-alphanumeric metacharacters must be escaped with a backslash to be used as ordinary characters within a regexp; alphanumeric metacharacters must be escaped with a backslash to NOT be used as ordinary characters. A really useful reference for Ruby's metacharacters can be found at the bottom of the page at Rubular.

The basic mechanisms of regexps are the same, wherever you may use them, but no other language has identical regexp syntax to Ruby's, so if you use, say, Python, you'll have to do a bit of research.
If you want to match any character "a"-"e" or "^", you can either escape the "^" with a backslash, like /[\^a-e]/, or place it after the beginning, like /[a-e^]/
Each individual backslash has to be escaped, so to match consecutive characters "\\", you'd need /\\\\/.
